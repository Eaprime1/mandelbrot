## 6. Naming Convention

*Eric's instinct, formalized:*

**Problem**: ALL CAPS is hard to type and creates visual noise where everything
looks equally important.

**Solution** (see `docs/NAMING_CONVENTION.md`):

| Use | Form | Example |
|-----|------|---------|
| Repos | lowercase | `mandelbrot`, `complexity` |
| Elevated designation | `}name{` | `}mandelbrot{` |
| Config folders | `.name/` | `.mandelbrot/` |
| Strong emphasis | ALL CAPS | `MANDELBROT` (sparingly) |
| Character names | Capitalized | Oaken, Valthram, Rema |

The curly bracket `}name{` form is already in use (see `}primehaven{` in
claude.md). Formalizing it as THE designation for "this is the concept,
not an instance" completes the pattern.

*Claude's addition*: The `}name{` form has a subtle mathematical reading —
`{x | condition}` is set notation. `}mandelbrot{` = "the set of mandelbrot" —
not one instance, but the whole concept-set that name encompasses.
